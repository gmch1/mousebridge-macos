# MouseBridge for Windows

Version: `0.1.0-preview`

MouseBridge for Windows is a small .NET 8/WinForms tray application for mouse
button mappings, mouse-only wheel control, and Logitech M750 L hardware DPI.
It has no accounts, network requests, telemetry, advertising, or automatic log
uploads.

> **Verification status:** the Windows project cross-compiles and includes a
> Windows CI/self-test workflow. Logitech Signature Plus M750 L Bluetooth input
> and HID++ DPI have **not yet been physically verified on Windows**. Do not
> describe this preview as production-ready or hardware-tested.

## Requirements and installation

- Windows 10 22H2 or Windows 11, x64 for the first preview package.
- [.NET 8 Desktop Runtime (x64)](https://dotnet.microsoft.com/download/dotnet/8.0).
- A normal user account; MouseBridge does not request administrator rights.

Extract the ZIP and run `MouseBridge.exe`. Closing the settings window keeps
the tray process running; use **Exit** in the tray menu to restore the original
DPI on a best-effort basis and stop MouseBridge.

The preview artifact is framework-dependent and unsigned. Its SHA-256 file is
generated by CI. Legal notices and corresponding-source information are in the
packaged `Legal` directory.

## Features

- Maps middle, back, and forward mouse buttons to shortcuts.
- Empty actions pass the original button through; `none` consumes it.
- Accepts `ctrl`, `shift`, `alt`, and `meta` (the Windows key); `cmd` is read as
  an alias and saved as `meta`.
- Reverses vertical and horizontal mouse wheel axes independently.
- Keeps raw wheel magnitude at `scrollLines=0`; for 1–20, normalizes a discrete
  vertical detent to the requested logical line count. High-resolution deltas
  are not quantized.
- Filters documented touch/pen-originated legacy mouse messages. Precision
  touchpad separation still requires physical regression testing.
- Enumerates every readable/writeable `VID_046D&PID_B02C` HID interface and
  accepts one only after HID++ Root reports Adjustable DPI feature `0x2201`.
- Reads device DPI values, applies the nearest supported value, serializes all
  HID++ requests with a 1.8-second timeout, reconnects at low frequency, and
  restores the connection-time DPI during normal exit when possible.
- Watches the configuration directory and keeps the last valid configuration
  if an external edit is invalid.

Logitech Bolt receivers, other M750 variants, and other Logitech mice are not
claimed as supported in this preview.

## Configuration and CLI

Configuration is stored at `%LOCALAPPDATA%\MouseBridge\config.json`. Logs are
stored at `%LOCALAPPDATA%\MouseBridge\MouseBridge.log`; default logs do not
record individual button, wheel, or keyboard events.

```powershell
MouseBridge.exe config path
MouseBridge.exe config get
MouseBridge.exe config set middle|back|forward <shortcut|none>
MouseBridge.exe config set reverse-vertical|reverse-horizontal <true|false>
MouseBridge.exe config set scroll-lines <0-20>
MouseBridge.exe config set dpi <400-4000>
MouseBridge.exe diagnose
MouseBridge.exe --self-test
```

Exit codes are 0 for success, 2 for invalid arguments, and 1 for runtime or
configuration errors. `diagnose` reports the live tray process when available,
HID interface accessibility, device-reported DPI state, paths, OS, and
architecture. Windows UIPI prevents `SendInput` from targeting applications
running at a higher integrity level; MouseBridge reports this without silently
requesting elevation.

## Build

No third-party NuGet runtime packages are used.

```powershell
dotnet restore platforms/windows/MouseBridge/MouseBridge.csproj
dotnet build platforms/windows/MouseBridge/MouseBridge.csproj -c Release --warnaserror
platforms/windows/MouseBridge/bin/Release/net8.0-windows/MouseBridge.exe --self-test
dotnet publish platforms/windows/MouseBridge/MouseBridge.csproj -c Release -r win-x64 --self-contained false
```

The built-in self-test covers shortcut parsing/canonicalization, scroll
direction and discrete/continuous behavior, configuration normalization and
atomic saving, injected-event rejection, HID++ response/error parsing, and DPI
list explicit/range/terminator/invalid vectors.

## Architecture

- `ConfigStore`: versioned JSON, atomic same-directory replace, debounced
  directory watcher, and last-valid fallback.
- `MouseHook`: `WH_MOUSE_LL`, paired button suppression, `LLMHF_INJECTED`
  filtering, and `SendInput` injection with a private marker.
- `HidDeviceEnumerator`: SetupAPI/HID enumeration, VID/PID and report-capability
  checks.
- `HidppController`: interface probing, serialized requests, disconnect-safe
  cancellation, DPI discovery/application/restoration.
- `SettingsForm` and `MouseBridgeContext`: WinForms settings, tray lifecycle,
  pause/resume, diagnostics, and single-instance activation.
